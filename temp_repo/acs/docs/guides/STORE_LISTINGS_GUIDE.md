# Руководство по работе с Store Listings

## Введение

В данном проекте используется унифицированная структура для управления материалами Google Play Store и Apple App Store. Все материалы хранятся в общей папке [`store_listings/shared/`](store_listings/shared/) для устранения дублирования и упрощения поддержки.

## Структура папок

```
store_listings/
├── shared/                    # Общие материалы для всех языков
│   ├── en/                   # Английский язык
│   │   ├── name.txt                           # Название приложения (оба стора)
│   │   ├── description.txt                    # Полное описание (оба стора)
│   │   ├── keywords.txt                       # Ключевые слова (App Store)
│   │   ├── subtitle_app_store.txt             # Подзаголовок (только App Store)
│   │   ├── short_description_google_play.txt   # Краткое описание (только Google Play)
│   │   └── promo_text_google_play.txt        # Промо-текст (только Google Play)
│   ├── ru/                   # Русский язык
│   ├── uk/                   # Украинский язык
│   ├── es/                   # Испанский язык
│   ├── de/                   # Немецкий язык
│   ├── fr/                   # Французский язык
│   ├── it/                   # Итальянский язык
│   ├── ko/                   # Корейский язык
│   └── ar/                   # Арабский язык
├── app_store/                # Устаревшая структура (сохранена для справки)
├── google_play/              # Устаревшая структура (сохранена для справки)
├── screenshots/              # Скриншоты для обоих сторов
├── metadata.json             # Метаданные приложения
└── README.md                # Основная документация
```

## Типы файлов и их назначение

### Общие файлы (используются обоими сторами)

#### name.txt
- **Назначение**: Название приложения
- **Ограничения**: 
  - App Store: max 30 символов
  - Google Play: max 50 символов
- **Рекомендация**: Использовать короткое название до 30 символов для совместимости

#### description.txt
- **Назначение**: Полное описание приложения
- **Ограничения**: max 4000 символов
- **Формат**: Текст с эмодзи для визуального привлечения внимания

#### keywords.txt
- **Назначение**: Ключевые слова для App Store
- **Ограничения**: max 100 символов
- **Формат**: Слова через запятую, без пробелов после запятых
- **Пример**: `cosmetic scanner,ingredient analyzer,beauty safety,skincare`

### Специфичные файлы для App Store

#### subtitle_app_store.txt
- **Назначение**: Подзаголовок под названием приложения
- **Ограничения**: max 30 символов
- **Отображение**: Под названием приложения в результатах поиска

### Специфичные файлы для Google Play

#### short_description_google_play.txt
- **Назначение**: Краткое описание приложения
- **Ограничения**: max 80 символов
- **Отображение**: В списке приложений и результатах поиска

#### promo_text_google_play.txt
- **Назначение**: Промо-текст вверху страницы приложения
- **Ограничения**: max 170 символов
- **Особенность**: Можно обновлять без выпуска новой версии приложения

## Процесс добавления нового языка

### Шаг 1: Создание папки языка

```bash
mkdir store_listings/shared/{lang_code}
```

Где `{lang_code}` - двухбуквенный код языка по стандарту ISO 639-1.

### Шаг 2: Копирование и перевод файлов

```bash
# Копирование файлов с английского языка
cp store_listings/shared/en/name.txt store_listings/shared/{lang_code}/name.txt
cp store_listings/shared/en/description.txt store_listings/shared/{lang_code}/description.txt
cp store_listings/shared/en/keywords.txt store_listings/shared/{lang_code}/keywords.txt
cp store_listings/shared/en/subtitle_app_store.txt store_listings/shared/{lang_code}/subtitle_app_store.txt
cp store_listings/shared/en/short_description_google_play.txt store_listings/shared/{lang_code}/short_description_google_play.txt
cp store_listings/shared/en/promo_text_google_play.txt store_listings/shared/{lang_code}/promo_text_google_play.txt
```

### Шаг 3: Перевод содержимого

Переведите содержимое всех файлов на новый язык, соблюдая ограничения по количеству символов.

### Шаг 4: Обновление документации

Обновите файл [`store_listings/README.md`](store_listings/README.md) и этот файл, добавив информацию о новом языке.

## Рекомендации по переводу

### Общие принципы

1. **Адапация, а не дословный перевод**: Учитывайте культурные особенности рынка
2. **SEO-оптимизация**: Используйте релевантные для рынка ключевые слова
3. **Соответствие ограничениям**: Всегда проверяйте количество символов
4. **Тестирование**: Проверьте, как текст выглядит в интерфейсе стора

### Специфика для разных языков

#### Русский и украинский языки
- Увеличение длины текста на 15-20% по сравнению с английским
- Использование кириллицы требует проверки отображения
- Учет культурных особенностей косметического рынка

#### Арабский язык
- Направление письма справа налево
- Визуальная длина текста может быть короче
- Необходима проверка корректного отображения в интерфейсе

#### Азиатские языки (корейский)
- Компактность текста - можно использовать больше информации
- Особенности косметического рынка Кореи
- Учет культурных предпочтений

## Процесс публикации

### App Store

1. Откройте App Store Connect
2. Перейдите в раздел "App Information"
3. Для каждого языка:
   - Название: `shared/{lang}/name.txt`
   - Подзаголовок: `shared/{lang}/subtitle_app_store.txt`
   - Описание: `shared/{lang}/description.txt`
   - Ключевые слова: `shared/{lang}/keywords.txt`

### Google Play

1. Откройте Google Play Console
2. Перейдите в раздел "Store listing"
3. Для каждого языка:
   - Название приложения: `shared/{lang}/name.txt`
   - Краткое описание: `shared/{lang}/short_description_google_play.txt`
   - Полное описание: `shared/{lang}/description.txt`
   - Промо-текст: `shared/{lang}/promo_text_google_play.txt`

## Автоматизация

### Скрипт для проверки ограничений

Можно создать простой скрипт для проверки соответствия ограничениям:

```python
import os

def check_file_limits(lang_code):
    base_path = f"store_listings/shared/{lang_code}"
    
    checks = [
        ("name.txt", 30, "App Store name limit"),
        ("name.txt", 50, "Google Play name limit"),
        ("description.txt", 4000, "Description limit"),
        ("keywords.txt", 100, "Keywords limit"),
        ("subtitle_app_store.txt", 30, "Subtitle limit"),
        ("short_description_google_play.txt", 80, "Short description limit"),
        ("promo_text_google_play.txt", 170, "Promo text limit")
    ]
    
    for filename, limit, description in checks:
        filepath = os.path.join(base_path, filename)
        if os.path.exists(filepath):
            with open(filepath, 'r', encoding='utf-8') as f:
                content = f.read().strip()
                length = len(content)
                status = "✅" if length <= limit else "❌"
                print(f"{status} {description}: {length}/{limit} ({filename})")

# Проверить все языки
for lang in ["en", "ru", "uk", "es", "de", "fr", "it", "ko", "ar"]:
    print(f"\nChecking {lang}:")
    check_file_limits(lang)
```

## Частые проблемы

### Проблема 1: Превышение ограничений по длине

**Решение**: Используйте скрипт проверки или вручную проверьте длину каждого файла. Сократите текст, сохраняя ключевые слова и смысл.

### Проблема 2: Некорректное отображение символов

**Решение**: Убедитесь, что файлы сохранены в кодировке UTF-8. Проверьте отображение специальных символов в интерфейсе стора.

### Проблема 3: Отсутствие перевода для нового языка

**Решение**: Убедитесь, что все файлы для нового языка созданы и переведены. Проверьте правильность названия папки языка.

### Проблема 4: Несоответствие перевода рынку

**Решение**: Проведите исследование целевого рынка. Адаптируйте текст под культурные особенности и ожидания пользователей.
